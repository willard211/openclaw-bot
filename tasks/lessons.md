# Lessons

## Process

- When the user provides persistent workflow constraints, codify them in project-local task files immediately instead of relying on conversational memory.
- For this repo, start non-simple tasks by checking `tasks/lessons.md`, then writing or updating `tasks/todo.md` before implementation.
- Use self-confirmation only when the task is low-risk and the plan is unambiguous; otherwise stop and get explicit user confirmation.
